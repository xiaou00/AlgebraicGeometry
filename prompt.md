# 输入参数

> 该文档是一个指令模板, 用于借助 AI agent 扩充和完善数学文档.

1. **数学输入格式**

- 文档采用Typst, 先阅读 [Typst官方文档](https://typst.app/docs) 了解语法.
- 阅读本地模板文件 `template.typ` 了解常用的数学符号.
- 对于行内公式, 使用 `$...$` 包裹; 对于独立行公式, 使用 `$ ... $` 包裹.

- 独立行公式的最后不需要任何符号, 写在一行内的全部用行内公式, 独立行公式必须单开一行. 下面的例子是错误的:

```typst
$ a + b $.
$ a + b. $
$ a + b $;
xxx $ a + b $, xxx /* 没有单开一行 */
```
- 统一使用半角标点.
- 尽可能多使用 typst 的符号系统, 例如用 `->` 代替 `arrow`.
- typst的 `_` 和 `^` 会自动识别连续段, 如果要加括号不能直接加在后面, 例如 `f_i(x)` 会把 `i(x)` 识别为下标, 正确写法是 `f_i (x)`.加空格.

2. **补充内容**

- 你需要根据上下文补充一些你觉得可以添加的内容.
- 纠正语法错误, 例如拼写错误, 语法错误, 逻辑错误等.
- 对于一些没有证明的命题, 你可以选择给出证明, 也可以选择不证明, 简短的证明或者详细的证明都可以, 你可以根据上下文来判断.
- 多添加例子和图表, 以便更好地说明问题.
- 添加一些能够助理解的东西, 例如一些注释, 解释, 说明, 直观的解释, 直观的图表等.
- 如果可以, 可以绘制一些 `svg` 图片. 但你最好保证不要绘制错误.
- 适当地对于某些容易混淆的概念, 进行一些解释和说明, 以便更好地理解.
- 对一些有难度的过程, 进一步拆解.
- 合理使用 `#remark` 来添加一些注释, 解释, 说明, 直观的解释.
